# Campaign Review Assistant

专题验收助手，面向网站专题产品的本地验收辅助工具。

## 项目定位

这个工具不是像素级设计还原检查器，而是帮助产品经理围绕三件事快速整理验收证据：

1. 页面展示与设计稿是否一致
2. 页面内容与业务方 / CMS 文案是否一致
3. 页面功能与需求文档 / UE 描述是否一致

## 当前能力

- 对话式入口：粘贴页面 URL、CMS 文案、需求/UE 描述或本地资料文件夹路径
- 自动移动端截图：使用本机 Chrome / Edge 采集页面长图
- 页面 DOM 证据采集：提取可点击元素、链接、图片、CSS 背景图和大致位置
- 三类自动检查：
  - 页面展示：检查设计稿/页面截图是否就绪，识别关键文案、入口、背景素材和位置线索
  - 页面内容：使用页面 DOM 文本 + 图片 OCR 文本对比业务方 / CMS 文案，并自动忽略明显的需求描述句
  - 页面功能：按需求/UE 描述匹配按钮、链接、跳转入口和 JS 点击证据，并支持自动点测
- 设计稿与页面截图证据区
- 视觉参考图：辅助发现明显模块、入口、背景差异
- CMS 文案 v0：使用页面 DOM 文本对比业务方文案
- 图片 OCR v1：使用 Tesseract.js + Sharp 预处理识别设计稿 / 页面截图中的图片化文字，并支持人工修正后重新参与 CMS 文案对比
- 功能点测 v0：用 Playwright 自动点击匹配到的入口，记录跳转、弹窗和页面变化
- 点测截图：功能点测会记录点击前 / 点击后截图，辅助复核跳转和弹窗状态
- 失败原因：点测会区分未找到入口、点击未命中、点击后无明显变化、异常失败等原因
- 本地资料搜索：扫描指定文件夹中的文档和图片资料，命中结果可一键纳入验收输入
- 资料预览：文本资料可直接预览并追加到验收输入，图片资料可一键设为设计稿
- 人工确认：每条检查结果可标记为通过、需修改、待确认或忽略，并支持填写备注
- 项目历史：可新建、保存、载入和删除本地验收项目，保留轻量验收证据
- 验收模板：内置网页移动端、小程序、游戏内嵌页、网吧内嵌页四类检查模板
- 本地保存：人工确认状态、备注、最终结论、智能总结和项目历史会保存到浏览器 localStorage
- 验收闭环：根据展示、内容、OCR、功能、资料和人工确认状态生成结论草稿
- 智能总结：本地生成风险判断、重点问题和建议下一步，不依赖云端模型
- 验收报告：一键复制或下载 Markdown 验收报告，报告包含 OCR 修正、资料摘录、点测前后截图和结构化失败原因
- 验收摘要与问题清单：问题清单会尊重人工确认结果

## 使用方式

在聊天框里可以直接粘贴：

```text
页面 URL
页面右上角要有下载游戏
中间要有预约按钮
背景图是同一个
购票按钮点击后跳转购票页
剑网3十七周年演唱会票正式开售
查看演唱会详情
```

也可以粘贴本地资料目录：

```text
D:\vscode\动效\docs 周年庆 预约 演唱会
```

如果路径里有空格，建议用引号包起来。

完成检查后，可以在“三类自动检查”中逐条确认：

```text
人工通过：确认该项无问题
需修改：需要进入问题清单和报告
待确认：暂时保留，不直接判定
忽略：从问题清单中移除
```

右侧“验收摘要”可以填写最终结论，并复制或下载 Markdown 报告。

## 当前边界

- 视觉检查不是像素级验收结论，只辅助判断关键模块、入口、背景和位置是否明显异常
- 图片 OCR 支持人工修正，但复杂中文艺术字、低对比度背景字仍可能误识别，关键上线文案仍建议人工复核
- 功能点测 v0 能识别入口并点击，能记录 URL 变化、弹窗和截图；登录态、复杂弹窗流程、多步骤任务链仍需要人工确认
- Markdown 报告会内嵌点测截图，报告文件会比纯文本报告更大
- CMS 文案对比优先基于页面 DOM 文本；运行图片 OCR 后会把 OCR 文本一起纳入对比
- 历史项目保存的是轻量证据，不会重复保存大型截图；需要长期归档时建议导出 Markdown 报告
- 智能总结当前是本地规则总结，不等同于大模型语义审稿，后续可接入 OpenAI 或本地模型
- PDF、DOCX、XLSX 目前可被资料搜索命中，但暂不直接解析全文预览

## 本地启动

安装依赖：

```bash
npm install
```

启动本地截图 / 资料搜索 API：

```bash
npm run api
```

启动前端：

```bash
npm run dev
```

默认地址：

```text
前端：http://127.0.0.1:5174
API：http://127.0.0.1:4317
```
